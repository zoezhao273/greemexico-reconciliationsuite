🔍 What gets checked (Validation Rules) ▼
Check 1

Duplicate UUIDs

Runs first. When a real UUID appears on more than one record, it tells two cases apart automatically: equal partial shares that sum within the invoice total (or a Bank Charges item, PPD payment) are treated as an amortized invoice — expected, with total/VAT/month relaxed. Full-amount repeats are flagged as possible duplicate reimbursement. Placeholder UUIDs (-, NA, free text for no-invoice items) and repeats within one record are never counted as duplicates.

Check 2

Valid UUID

A UUID is valid only when both conditions hold: it exists in the CFDI file and the matched invoice's Estatus is exactly Vigente. A stray : prefix, spaces, or unusual dashes are tolerated; anything still unmatched is flagged as not found. A UUID that is found but is Cancelado (or any non-Vigente status) is flagged as invalid. A line whose Material = Non-Deductible Expense is exempt from the not-found case only — these legitimately have no CFDI.

Check 3

Posting month

NCC DocDate month must equal CFDI Emisión month. Run only when the record is DocNo starting with AP and A/PType = CA Reimbursement, or when the supplier contains PETTY CASH.

Check 4

Total amount

NCC AmtIncl.VAT(OC) must equal CFDI Total Original XML to the cent (no tolerance). A split-by-expense-type invoice is summed within the record first. If booked below the invoice it reads "Please verify the bank payment amount"; if above, "does not match".

Check 5

VAT amount

NCC VATAmt(FC) must equal CFDI IVA to the cent (summed per invoice within a record). Exception: for Travel – Meals, VAT equal to 8.5% of the CFDI IVA (±0.01) is accepted — SAT allows only 8.5% deductible for meals within 50 km. If that 8.5% pattern matches but Material ≠ Travel – Meals, the Total/VAT mismatch is still flagged, with the reason Meal ≤ 50 KM but incorrect material (the Material column keeps the NCC value). Also skipped for PY non-MXN lines (settled at the payment-date FX rate).

Check 6

Exchange rate

Only when NCC Currency is not MXN: NCC Ex.Rate must equal CFDI Tipo Cambio. Skipped for PY payment documents, whose foreign-currency lines use the payment-date rate rather than the invoice rate.

Check 7

Withholding tax

For lines where VATCode = PresetWHTL0 Donot used, the absolute value of AmtIncl.VAT(OC) must equal CFDI IVA Retenido + ISR Retenido.

Note

Exemptions

Skipped entirely: records whose supplier is GREE AIRCONDITIONING MEXICO or CDMX GOV (salary / tax), and PY documents whose PmtType is Cash Advance Payment or Cash Avance Return (not invoice related). The issuer-RFC check is also not run.

About CFDI dates. The Emisión column is read with file-level format sniffing: the format is detected once from the first dated row and reused for the whole file. Native Excel dates and serial numbers are read directly; text dates are split on / or - and locked as year-first (YYYY-MM-DD) when the first part is 4 digits, or day-first (DD-MM-YYYY) when the last part is 4 digits. Only the year and month are used for the posting-month check.

How to use

  • 1
    Export the monthly NCC Bank Journal (.xlsx) from your accounting system. Export the ICBC Bank Statement (.csv, UTF-16) and optionally the Santander Bank Statement (.csv, UTF-8) from each bank portal.
  • 2
    Upload both the NCC Bank Journal and a Bank Statement — both are required. For the Bank Statement, upload either the ICBC file or the Santander file, not both at the same time. Uploading a second bank file will automatically clear the first.
  • 3
    Click Run Reconciliation. The tool auto-detects formats, skips subtotal rows, sorts by amount, and compares line by line.
  • 4
    Preview the discrepancies on screen, then click Export to Excel to save the reconciliation report with two worksheets.

    Yellow highlight = entry missing on one side. Green = matched. The Note column explains each discrepancy.

How it works

  • 1
    Upload all three workbooks (.xlsx): NCC AP — CA Reimbursement, NCC PY — Pmt Doc For SCM, and the Received CFDI file. Each card has its own Clear button to swap a single file without resetting the others.
  • 2
    The tool reads the itemized bodys detail table from each NCC file and keeps only rows whose UUID is a well-formed 36-character invoice folio (8-4-4-4-12). Blank values and placeholders such as -, tip, or NA are skipped; spacing and look-alike dashes are normalized first so genuinely valid folios are recovered.
  • 3
    VAT (VATAmt(FC)) is summed per UUID across both NCC files, then each UUID is left-joined to the CFDI file to pull its Emisor RFC and Estatus. A UUID is fully valid only when it exists in the CFDI file and its status is exactly Vigente.
  • 4
    Review the on-screen report — flagged (non-Vigente) rows are tinted — then click Export to Excel to export the four-column DIOT report: VAT, Emisor RFC, UUID, Estatus.
